# コミットの方針

指摘が無くコミットに進む際に、これからの変更とブランチの未マージのコミットをどう整えるかを判断するための方針。

- メインブランチに取り込まれていない変更のコミットを、適切な粒度に整える。
- PR には意味のある単位のコミットを含める。
- typo 修正・リファクタリング・テストコード追加などのマイナーな修正は、既存のコミットに amend する。
- 追加方針や方針変更により作業ブランチの内容が大きく変わった場合は、squash して最終的な提出コミットにまとめる。方針変更の過程はコミットに残さない。
- 最終的な提出物が意味的に独立しており、コミットを分割するのが適切な場合にのみ、コミットを追加する。
- コミットメッセージおよび PR のタイトル・本文の言語は、CLAUDE.md に指定があればそれに従う。指定がなければ英語で書く。
